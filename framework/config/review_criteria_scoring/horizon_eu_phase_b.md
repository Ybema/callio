# Phase B: Work Package Analysis - Review Criteria & Scoring
**Funding Type**: HORIZON Europe  
**Phase**: B (Work Package Analysis)  
**Purpose**: Assess individual WPs using two-layer criteria (External + Internal)

## Two-Layer Assessment Framework

### LAYER A: External (Call-Specific) Criteria (50%)
*Load from call document - these change per call*

#### A1. Call Scope Alignment (15%)
- TRL expectations met
- Value chain integration present
- Sustainability aspects covered
- Regional inclusion addressed

#### A2. Expected Outcomes Contribution (20%)
- Clear mapping to EO1-EO4
- Quantified contributions where possible
- Strong evidence of delivery

#### A3. SRIA Strategic Priorities (15%)
- Alignment with strategic priorities
- KPI contributions identified
- Mission objectives addressed

### LAYER B: Internal (Fixed) Criteria (50%)
*Consistent across all HORIZON proposals*

#### B1. WP Structure & Logic (20%)
**What to Check**:
- WP Objectives clearly defined
- Tasks (T1.1, T1.2) logically structured
- Deliverables mapped to Tasks
- Milestones meaningful and timed

**Readiness Scale**:
- **Level 5 - Final**: Text matches Part B format (concise, clear, consistent)
- **Level 4 - Validation**: Full WP logic present (Risks, Milestones, dependencies)
- **Level 3 - Alignment**: Internal logic clear (Tasks→Deliverables→Objectives→Outcomes)
- **Level 2 - Content**: Core structure exists (Objectives, Tasks, Deliverables listed)
- **Level 1 - Skeleton**: WP has only basics (Objectives + link to Outcomes)

#### B2. LFA Integration (15%)
- WP Objectives map to project Outcomes
- Deliverables align with project Outputs
- Tasks cover required Activities
- Cross-WP dependencies identified

#### B3. Quality Standards (10%)
- Consistent terminology (SO, O, D, T, M)
- No contradictions across WPs
- Proper grammar and style
- EU-evaluator friendly tone

#### B4. Implementation Readiness (5%)
- Partner roles clear
- Resource allocation realistic
- Risk mitigation credible
- Timeline achievable

## WP Readiness Assessment

### Individual WP Score
**Calculation**: (Layer A × 50%) + (Layer B × 50%)

### Project-Level WP Readiness
**Calculation**: Average of all WP scores

**Interpretation**:
- **Level 5 (4.5-5.0)**: WPs are submission-ready, proceed to Phase C
- **Level 4 (3.5-4.4)**: WPs are strong, minor refinements needed
- **Level 3 (2.5-3.4)**: WPs need focused improvements
- **Level 2 (1.5-2.4)**: WPs need significant development
- **Level 1 (1.0-1.4)**: WPs need major restructuring

## Cross-WP Integration Check
- Dependencies properly mapped
- No gaps in coverage
- No unnecessary overlaps
- Consistent narrative across WPs
